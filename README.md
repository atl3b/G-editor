# G-editor

一款从零构建的轻量级桌面文本编辑器，使用 C# / .NET 8 / WPF 技术栈开发。

所有代码均为原创或来自与 MIT 兼容的许可证。严禁直接拷贝或近似改写任何采用 GPL 及其衍生许可证的开源项目源码。

## 设计目标

- **轻量可扩展**：模块化架构，核心逻辑与 UI 层严格分离，便于测试和迭代
- **渐进式开发**：以 MVP 为起点，逐步扩展功能
- **开发者友好**：清晰的分层设计，核心层不依赖 UI 框架，可独立测试

## MVP 功能范围

- 文件操作：新建、打开、保存、另存为
- 多标签页编辑
- 行号显示、当前行高亮、自动缩进
- 撤销 / 重做
- 查找与替换（支持正则表达式）
- 状态栏（光标位置、编码、换行符类型）
- 基础语法高亮（C#、JSON、XML）
- 换行符识别与保留（CRLF / LF / CR）

## 项目结构

```
G-editor/
├── src/
│   ├── GEditor.Core/        # 核心逻辑（文档模型、编辑命令、文件 IO、搜索）
│   ├── GEditor.Syntax/      # 语法高亮引擎（无 UI 依赖）
│   └── GEditor.App/         # WPF 应用宿主
└── tests/
    └── GEditor.Tests/       # 单元测试
```

`GEditor.Core` 和 `GEditor.Syntax` 不引用任何 UI 包，可独立于 WPF 进行测试和复用。

## 编码与文本处理策略

### 内部编码

编辑器内部统一使用 .NET 字符串的原生编码 UTF-16。

### 打开与保存

- **默认编码**：UTF-8（无 BOM）
- **BOM 检测**：打开文件时优先通过 BOM 头识别编码（UTF-8 BOM、UTF-16 LE/BE）；无 BOM 时回退为 UTF-8 启发式检测，最终兜底至系统默认编码
- **编码选择**：保存时可选指定编码；未指定时保持打开时的编码

### 换行符处理

- 打开文件时自动检测换行风格（CRLF / LF / CR）
- 编辑过程中内部统一处理，不丢失换行符信息
- 保存时默认保留原文件换行风格，可通过 UI 切换
